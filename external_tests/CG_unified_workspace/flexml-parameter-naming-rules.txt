Similar to the robotworkspace/tagging-rules.txt which is used to organize (agree) necessary unification of tag name usage
this file is used to organize necessary unification of flexXml parameter (column) name usage within all CG TCs.
Without this centralized administration it would not be possible to reliably create a robot test suite that
combines multiple CG TCs of different authors, since it would not be ensured that the meaning of a single 
parameter (e.g vlrAddress) is identical in the context of different CG TCs.

When a CG TC author creates a new TC and therefore needs some fleXml parameters, he first has to
search the list below whether or not a corresponding parameter is already defined. If so he 
MUST use the pre-defined parameter name. Otherwise if no corresponding parameter is defined for his purpose,
then he has to create an additional list entry with his prefered parameter name, so that in future other authors
will use his definition if they have a similar need.
